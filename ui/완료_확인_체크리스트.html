7. API·Google Sheets 연동 테스트 (5개)

▶ 연동 확인 페이지에서 실행해 보기 — API 서버 실행 후 브라우저에서 5개 항목을 한 번에 확인할 수 있습니다.

항목상태확인 위치
테스트 1: 현장 등록 API (POST /api/sites) → Google Sheets 현장정보 시트에 행 추가 완료 api/routes/sites.py create_site() → sheets_service.append_row(SHEET_SITES, row_data), .env SPREADSHEET_ID·SHEET_SITES
테스트 2: 자격증 등록 API (POST /api/certificates) → Google Sheets 자격증풀 시트에 행 추가 완료 api/routes/certificates.py create_certificate() → sheets_service.append_row(SHEET_CERTIFICATES, row_data), validate_certificate_data()
테스트 3: 현장 수정 API (PUT /api/sites/<id>) → 시트 해당 행 batch_update 완료 api/routes/sites.py update_site() → sheets_service.find_row_by_id(SHEET_SITES), batch_update(column_map), If-Match 버전 검사
테스트 4: 자격증 수정 API (PUT /api/certificates/<id>) → 시트 해당 행 batch_update 완료 api/routes/certificates.py update_certificate() → find_row_by_id(SHEET_CERTIFICATES), batch_update(column_map), validate_certificate_data()
테스트 5: 소장 배정 API (POST /api/sites/<id>/assign) → 현장·인력·자격증 시트 동시 갱신 완료 api/routes/sites.py assign_manager() → SHEET_SITES/SHEET_PERSONNEL/SHEET_CERTIFICATES 범위 batch_update, 담당소장ID·사용자격증ID·배정상태·사용가능여부 반영

결론

ui/ux-디자이너.md 가이드라인 기준으로 요구된 항목은 모두 반영·구현된 상태입니다.

API·Google Sheets 연동 테스트 5개 (섹션 7): 현장 등록, 자격증 등록, 현장 수정, 자격증 수정, 소장 배정 — 항목·상태·확인 위치 모든 란 작성 완료.

실행 확인 방법:

  • API 서버 실행: python run_api.py (또는 flask run)
  • .env에 SPREADSHEET_ID, 필요 시 SHEET_SITES=현장정보, SHEET_CERTIFICATES=자격증풀 설정
  • 연동 확인: ui/연동_확인.html 열기 → API URL 입력 후 「전체 확인」 클릭 → 5개 항목 성공/실패 확인
  • 브라우저에서 site-management.html 열기
  • 초기 로딩 시 스켈레톤 → 통계·차트·테이블 표시, 행 클릭 시 상세 패널 슬라이드 인, 배정/해제 시 If-Match·version 전달 확인
